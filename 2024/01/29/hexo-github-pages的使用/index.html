hexo+github pages的使用

2024-01-29 Comments

安装Node.js

我第一次做的时候可能是只做了add和use，导致打开一个新的powershell的时候，无法使用hexo命令（因为node -v显示不出，node.js没有成功安装）

hexo安装、初始化、发布文章、本地网站（hexo本地服务器）打开、部署基本

这里git config其实配置的是~/.gitconfig文件，和下文的git config –global user.name(user.email)配置的是同一个文件（c/用户/huawei/.gitconfig文件）

发布文章用hexo new "文章的题目"，文章在source/_posts，图片在source/images

本地网站打开用hexo server

部署用hexo clean、hexo generate、hexo deploy

hexo本地网页的custom设置

我选择了作者cofess的theme主题，仓库及中文文档的地址是：

https://github.com/cofess/hexo-theme-pure/blob/master/README.cn.md

首先拷贝仓库，则在hexo的文件夹下会多出一个themes文件夹

hexo s表示hexo server，可以在本地看到网站

修改themes下的_config.yml：

个人简介这边images/opengl.jpg可以改头像

location和author, autor_title直接改掉

follow改掉，点击头像就会跳转github的主页

social下面把weibo~rss给注释掉，就不会显示这几个社交媒体的图标，social下面的github改掉，则点击github图标也会跳转github的主页

menu这边，可以添加或删除侧边栏页面导航

主题这边，如果什么都不写只有skin: ，则是白色主题，否则可以设置黑色、绿色等主题

hexo部署（发布到github pages）

我采用的是官网的一键部署，注意，全程不需要git init、git remote add+在hexo这个文件夹下本身也不能够使用git命令：

首先，在github创建仓库，仓库的名字是自己的github账号名字+github.io，选择public，不需要readme.md，剩下的跟着hexo官网教程的一键部署就行：

部署 | Hexo

或者参考hexo-deployment-git这个一键部署工具的仓库：

https://github.com/hexojs/hexo-deployer-git

hexo generate会创建public文件夹，也就是全部会被push到github仓库的文件

在hexo文件夹的_config.yml应该设置的是url以及deploy

每次传的时候主要是下面三条命令：

  • hexo clean
  • hexo g（hexo generate）
  • hexo deploy

部署一直失败其实可以看一下github最近的公告，不过我的原因并不是因为github pages在维修，公告地址如下：

https://www.githubstatus.com/

部署时有两个关键问题：

关键问题一

背景：

我创建了一个新的谷歌邮箱

然后基于这个邮箱创建了一个新的github账号

然后基于这个github账号创建了一个github pages

然后我想push东西到这个github pages

但是一直提示我denied to xxx695470536（原来的github账号）

其实一共有三个地方需要改：

  • ~/.ssh文件夹下的公钥和私钥都需要修改，有以下注意事项：

    • 使用rsa算法（有rsa和ed25519两种算法）

    • 首先要确保~/.ssh/id_rsa和~/.ssh/id_rsa.pub两个文件删除，否则覆盖写入的时候会有问题

    • 私钥设置时，使用git bash（windows powershell不行）开启ssh-agent（下图），然后再ssh-add ~/.ssh/id_rsa

      ssh -vT git@github.com会连不上，timed out，但是不影响，不需要连上

    • 公钥设置时，拷贝id_rsa.pub到github官网的settings->ssh and gpg keys，这里需要公钥的名字，名字可以随便取，我取了ssh-rsa

    公钥和私钥添加可以看github官网，不过我没有用ed25519的算法，用了rsa算法：
    https://docs.github.com/en/authentication/connecting-to-github-with-ssh/adding-a-new-ssh-key-to-your-github-account

  • win搜索：凭据管理器

    然后找到git，然后把账号改成qianlipages，密码不是对应github账号的登陆密码，而是对应qianlipages的github账号的settings->developer settings->personal access tokens（从github官网生成）

  • 设置邮箱和用户名

    git config –global user.name “qianlicnpages”

    git config –global user.email “qianlicn2023@gmail.com“

    这里的用户名和邮箱其实和~./gitconfig里的内容是一样的

    而如果是希望仓库的用户名和邮箱修改，则在仓库输入：

    git config user.name “qianlicnpages”

    git config user.email “qianlicn2023@gmail.com“

当这个问题解决以后，push就能成功了，此时打开自己的仓库地址是能看到commit的记录的

准确来说当我改完凭据管理器，空的仓库立刻就有文件了，虽然显示的是十几分钟前的更新

关键问题二

当我push成功，仓库也能看到我push的文件了，但是qianlipages.github.io看不到网页

chatgpt告诉我通过仓库的settings->pages选择gh-pages，刚刚设置完，立刻就看到settings->pages这边显示了我的github pages的地址！此时仓库的settings->general的default branch也会变成gh-pages

在选择gh-pages之前的选项是None，build and deployment上边是不会显示your site is live at这一段的

在选择gh-pages之后立刻就显示出来了，对应的qianlipages.github.com也立刻能刷出来

如果设置了custom domain，另外需要CNAME文件，表示另外的域名(chatgpt提到的网页404的原因之一)

其实两个最关键的问题都是chatgpt帮我解决的。。。chatgpt真是太造福人类了！
